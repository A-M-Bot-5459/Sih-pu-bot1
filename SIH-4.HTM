😈 SIH25012 — First-Prize Playbook

Layered reveal, step-by-step instructions and show-off lines — built to win.

Day 1 — Setup & Strategy

Day 1 — Setup & Strategy
3 steps
Quick action: name one leader, split tasks into coding/design/pitch/test. Keep roles clear.
Create a GitHub repo, add README, create branches feature/main. Commit often with clear messages.
Underline what the problem *really* asks; identify constraints and expected outputs.
Search past winners/case studies to spot gaps. Your edge = exploit gap + simple UX.
Paper or Figma: draw the minimum screens that show the core value instantly.

Day 2 — Build & Break

Day 2 — Build & Break
3 steps
Implement the core value — the rest is polish. Use libraries to speed up.
Record a short backup demo video (30–60s) in case live demo fails.
Ensure the main flow completes end-to-end at least once with realistic data.
Practice handovers, demo timing, and Q&A. One person plays 'tough judge'.

Final Pitch — Showtime

Final Pitch — Showtime
3 steps
Less text, clear icons, 1 message per slide. Judges skim — make it obvious.
Short script: start, few clicks to show core value, finish. Practice until smooth.
Finish with a line that projects scale & impact. Example: “We can reach 1M users in 2 years.”
